Panduan Simulasi Sistem Kontrol Kecepatan Boat di Arduino

Panduan ini berlaku untuk semua metode kontrol kecepatan boat yang telah diimplementasikan (Fuzzy Sugeno, Fuzzy PID, dan Neural Network).

Persiapan Hardware dan Software

Peralatan yang Dibutuhkan:

  • Board Arduino (Arduino Uno, Mega, Nano, atau ESP32)
  • Kabel USB
  • Komputer dengan Arduino IDE terinstal

Persiapan Software:

  1. Download dan instal Arduino IDE versi terbaru
  2. Salin kode program sistem kontrol kecepatan boat yang ingin disimulasikan (Fuzzy Sugeno, Fuzzy PID, atau Neural Network)

Langkah-langkah Upload Program ke Arduino

1 Hubungkan Arduino ke Komputer

Gunakan kabel USB untuk menghubungkan Arduino ke komputer. Pastikan komputer dapat mendeteksi Arduino.

2 Buka Arduino IDE

Jalankan Arduino IDE yang telah terinstal pada komputer Anda.

3 Buat Project Baru

Klik File > New untuk membuat project baru. Halaman editor akan terbuka dengan kode template dasar.

4 Salin dan Tempel Kode Program

Hapus kode template yang ada dan salin (copy-paste) kode program sistem kontrol kecepatan boat yang ingin Anda simulasikan.

5 Pilih Board Arduino

Klik menu Tools > Board dan pilih jenis Arduino yang Anda gunakan, misalnya "Arduino Uno" atau "ESP32 Dev Module".

6 Pilih Port Serial

Klik menu Tools > Port dan pilih port serial tempat Arduino terhubung, misalnya "COM3" (Windows) atau "/dev/ttyUSB0" (Linux/Mac).

7 Verifikasi (Compile) Program

Klik tombol "Verify" (ikon centang) atau tekan Ctrl+R untuk memverifikasi kode. Tunggu hingga proses kompilasi selesai tanpa error.

8 Upload Program ke Arduino

Klik tombol "Upload" (ikon panah ke kanan) atau tekan Ctrl+U untuk mengunggah program ke Arduino. Tunggu hingga proses upload selesai.

Catatan: Jika terjadi error saat upload, periksa kembali pemilihan board dan port, serta pastikan tidak ada aplikasi lain yang menggunakan port serial tersebut.

Simulasi dan Pengujian Program

Menggunakan Serial Monitor

1 Buka Serial Monitor

Setelah berhasil upload program, buka Serial Monitor dengan mengklik ikon kaca pembesar di pojok kanan atas IDE atau tekan Ctrl+Shift+M.

2 Atur Baud Rate

Pastikan Baud Rate di Serial Monitor diatur ke 9600 (sesuai dengan Serial.begin(9600) di kode program).

3 Kirim Input Data

Masukkan input dalam format target,actual ke kolom input di bagian atas Serial Monitor, kemudian klik "Send" atau tekan Enter.

Contoh input: 50,45 untuk target kecepatan 50 km/h dan kecepatan aktual 45 km/h.

Contoh Pengujian

Berikut adalah beberapa skenario pengujian yang bisa Anda coba:

Input Skenario Ekspektasi
50,45 Kecepatan terlalu lambat (Error positif kecil) RPM akan meningkat
50,60 Kecepatan terlalu cepat (Error negatif sedang) RPM akan menurun
50,50 Kecepatan tepat sasaran (Error nol) RPM akan stabil
50,30 Kecepatan jauh lebih lambat (Error positif besar) RPM akan meningkat signifikan
50,80 Kecepatan jauh lebih cepat (Error negatif besar) RPM akan menurun signifikan
0,50 Ingin berhenti dari kecepatan tinggi RPM akan turun drastis

Penggunaan Mode Debug

Semua implementasi sistem kontrol memiliki mode debug untuk melihat nilai-nilai internal saat proses fuzzy atau neural network bekerja.

  • Untuk mengaktifkan/menonaktifkan mode debug, tambahkan kata debug setelah nilai aktual, contoh: 50,45,debug
  • Saat mode debug aktif, sistem akan menampilkan informasi lebih detail tentang proses perhitungan internal.

Mode debug berguna untuk memahami cara kerja sistem dan membantu proses tuning parameter jika diperlukan.

Perbandingan Antar Metode

Anda dapat mencoba ketiga metode kontrol (Fuzzy Sugeno, Fuzzy PID, dan Neural Network) dengan skenario input yang sama untuk membandingkan keluaran dan karakteristik masing-masing metode.

Metode Karakteristik Kelebihan Kelemahan
Fuzzy Sugeno Output langsung dari fungsi linear
  • Implementasi sederhana
  • Komputasi efisien
  • Mudah dikonfigurasi
  • Tidak memiliki "memori" (tidak mengakumulasi error)
  • Tidak mengantisipasi perubahan (tidak menghitung derivative)
Fuzzy PID Tuning parameter PID secara dinamis
  • Memiliki komponen integral untuk mengatasi steady-state error
  • Memiliki komponen derivative untuk mengantisipasi perubahan
  • Lebih responsif terhadap dinamika sistem
  • Implementasi lebih kompleks
  • Membutuhkan tuning yang lebih teliti
  • Komputasi lebih berat
Neural Network Training dari data sampel
  • Dapat "belajar" dari data historis
  • Potensial untuk adaptasi yang lebih baik
  • Dapat menangani hubungan non-linear yang kompleks
  • Butuh data training yang representatif
  • Proses training menambah kompleksitas
  • Hasil lebih sulit diprediksi secara intuitif

Troubleshooting

Masalah Umum dan Solusi

Masalah Kemungkinan Penyebab Solusi
Error saat kompilasi Kesalahan sintaks pada kode Periksa pesan error dan perbaiki sintaks kode
Error saat upload Board atau port tidak cocok Periksa pemilihan board dan port
Tidak ada respons di Serial Monitor Baud rate tidak cocok Sesuaikan baud rate di Serial Monitor dengan kode (9600)
Format input salah Input tidak sesuai format Gunakan format yang benar: target,actual
Output tidak masuk akal Error dalam perhitungan atau parameter Aktifkan mode debug untuk melihat proses perhitungan detail

Pengembangan Lebih Lanjut

Setelah berhasil mensimulasikan program, Anda dapat melakukan pengembangan lebih lanjut:

  1. Tuning Parameter - Sesuaikan parameter fungsi keanggotaan atau nilai-nilai konstan untuk mengoptimalkan respons sistem
  2. Penambahan Input - Tambahkan input lain seperti akselerasi atau variabel lingkungan (angin, arus air)
  3. Integrasi dengan Sensor - Hubungkan dengan sensor kecepatan nyata seperti GPS atau flow meter
  4. Interfacing dengan Aktuator - Hubungkan output dengan aktuator seperti ESC (Electronic Speed Controller) untuk motor
  5. Visualisasi - Tambahkan fitur visualisasi respons sistem, misalnya dengan grafik atau dashboard

Untuk implementasi pada boat sungguhan, disarankan untuk menambahkan fitur keamanan seperti batas waktu, limit switch, atau sistem fail-safe untuk mencegah kondisi yang tidak diinginkan.

Kesimpulan

Simulasi sistem kontrol kecepatan boat di Arduino merupakan langkah awal yang baik untuk memahami berbagai metode kontrol cerdas seperti Fuzzy Logic (Sugeno dan PID) serta Neural Network. Melalui simulasi, Anda dapat mempelajari karakteristik masing-masing metode dan memilih yang paling sesuai dengan kebutuhan aplikasi Anda.

Penggunaan Arduino sebagai platform prototyping mempermudah eksperimen dan pengembangan lebih lanjut sebelum implementasi pada sistem nyata. Dengan demikian, risiko dan biaya pengembangan dapat diminimalkan sementara kualitas dan keandalan sistem tetap terjaga.
